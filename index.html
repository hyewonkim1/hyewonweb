Section : Skills

HTML

웹 표준과 웹 접근성을 준수하는 HTML 정보 구조 설계로 웹사이트를 구축하거나 수정할 수 있습니다.

CSS

웹 표준을 준수하는 CSS 코딩으로 크로스브라우징(Cross Browsing)을 위해 안정적인 CSS 속성을 사용해서 HTML 엘리먼트를 배치하고 디자인하거나 수정할 수 있습니다.

JS

네비게이션, 탭 메뉴, 어코디언 등은 플러그인을 사용하지 않고 제이쿼리 하드 코딩으로 제작합니다. 필수적인 자바스크립트 활용은 가능하며 주로 제이쿼리와 병행해서 활용합니다.

React

기본적인 Router, Redux를 사용하여 웹페이지 만들기 가능합니다.

Responsive

반응형 웹 제작에 있어서 크로스브라우징(Cross Browsing)을 위해 안정적인 기존 방식인 float, position, Media Query를 활용한 반응형 웹 제작 경험을 가지고 있습니다.

SCSS(SASS)

기본적인 SCSS사용 및 mixin함수 사용이 가능하며, 전반적인 코드 이해 가능하여 SCSS로 진행되는 프로젝트인 경우 빠르게 적응할 수 있는 준비가 되어 있습니다.

Github

Git 명령어를 사용하여 Github와 동기화 하여 버전관리 및 협업이 가능합니다.

Ps, XD

포토샵을 이용하여 기본적인 이미지 편집이 가능합니다.
XD를 활용하여 12칼럼을 이용한 웹사이트 시안 작성이 가능합니다.

Section : Epilogue

thank you for reading
  • 퍼블리싱 원칙과 기본기 폴더 구조, 경로를 체계적으로 정리하고, 파일 및 폴더 네이밍에 규칙을 두어 효율적인 업무가 가능합니다. 퍼블리싱 작업 시, 한 눈에 알아볼 수 있도록 적 절한 주석을 달고, 가독성을 높이기 위해 인덴트에 신경 쓰면서 스스로의 작업 효율 뿐 아니라, 협업하는 동료까지 생각하는 기본기를 갖추었습니다.
  • 체계적인 퍼블리싱 퍼블리싱 작업에 앞서 와이어프레임, HTML 구조설계를 먼저 진행하여 꼼꼼하고 체계적으로 작업을 하고 있습니다. 이는 프로젝트를 시행하 기 전에 다양한 계획을 설정하여 혹시 모를 결과의 가능성을 대비하기 위함입니다. 이러한 준비된 행동들은 저에게 ‘체계적이며 성실하다’라는 수식어를 주었고, 나아가 더욱 완벽한 결과물을 만들어내는 데 도움을 주었습니다.
  • 하드코딩을 원칙으로 기능성을 제외한 모든 작업은 하드코딩을 원칙으로 순수 CSS 작업을 합니다. 레이아웃 작업 시, Frameworksd 뿐 아니라 Position, Float 기존방식, CSS Flex와 CSS Grid를 용이하게 사용합니다. 직접 모든 코딩의 구조를 설계하고 속성을 입력하기 때문에 후에 유지보수 및 수정 시에도 어렵지 않게 작업할 수 있습니다.
  • 준비된 성실함 사람들은 저에게 '적응이 빠르다.'라고 자주 이야기합니다. 이는 제가 스스로 발전해가는 과정을 좋아하기 때문입니다. 자기객관화를 통해 부족한 점을 찾아 개선하고 더 나은 결과물을 만들어내기 위해 노력합니다. 현실에 안주하지 않고 언제나 배우고 빠르게 발전해나가는 퍼블리셔가 되겠습니다.
